Cómo funciona el decodificador

El decodificador usa la Web Audio API del navegador para procesar el audio sin instalaciones adicionales. El proceso ocurre en varias etapas:

Audio de entrada
→
FFT por frecuencias
→
Detección de tono Morse
→
Análisis de energía
→
Segmentación puntos/guiones
→
Clasificación de duraciones
→
Texto decodificado

El sistema evalúa múltiples candidatos de frecuencia y selecciona el tono con mayor energía, mayor repetición y patrones ON/OFF consistentes. Esto permite ignorar ruido de fondo e interferencias moderadas.

Cómo usar el decodificador

  1. Subir archivo: haz clic en la zona de carga y selecciona tu archivo de audio, o arrástralo directamente. Acepta MP3, WAV, OGG, M4A y otros formatos comunes.

  2. Grabar micrófono: alternativamente, activa el micrófono para analizar audio en tiempo real. Útil para descifrar transmisiones de radio o entrenamiento con llave.

  3. Analizar: el decodificador procesa el audio y muestra la secuencia Morse detectada junto con el texto resultante. El espectrograma muestra el análisis de frecuencias en detalle.

  4. Revisar resultados: el resumen de análisis indica la frecuencia del tono detectado, la velocidad estimada en WPM y la confianza en la decodificación.

Consejos para mejores resultados

  • Usa grabaciones limpias con un solo tono Morse. El decodificador funciona mejor con señales CW puras.
  • La velocidad ideal para decodificación automática está entre 10 y 30 WPM. Velocidades muy altas o muy bajas pueden reducir la precisión.
  • Si hay mucho ruido de fondo, intenta con una versión del audio filtrada en un editor externo antes de subir.
  • Para grabaciones de micrófono, acerca el micrófono al altavoz de radio o usa un cable de audio directo para evitar ruido ambiental.
  • El espectrograma en tiempo real ayuda a confirmar que el sistema está detectando la frecuencia correcta del tono Morse.

Privacidad del audio

Tu audio nunca sale de tu dispositivo. A diferencia de servicios en la nube, codigomorse.link no sube el archivo a ningún servidor. Todo el procesamiento se realiza localmente mediante la Web Audio API estándar del navegador.

Esto también significa que el decodificador funciona sin conexión a internet una vez que la página ha cargado completamente.
